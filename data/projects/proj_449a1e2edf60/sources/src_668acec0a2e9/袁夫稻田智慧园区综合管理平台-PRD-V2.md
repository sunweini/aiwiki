修订历史

一、项目背景与目标
1.1 项目背景
袁夫稻田（YUANFU RICE FIELD）是以稻田文化为核心，集生态农业、休闲旅游、餐饮体验于一体的田园综合体品牌。旗下已布局黄梅、武汉等多个园区，每个园区内运营多家商户（火车餐厅、树下咖啡、稻田手作坊等），面向终端消费者提供餐饮、体验、活动等综合服务。
品牌目前使用的自主开发运营管理平台已积累了一定的业务数据，但随着业务规模的扩大与精细化运营的需求升级，原有系统在以下方面已无法满足要求：
（1）架构层面：仅支持单园区管理，无法同时支撑总部 - 园区 - 商户三层组织架构下的统一管控与独立运营。
（2）功能层面：缺乏数据看板、客流统计、活动推送、闸机管理、会员储值与亮码支付等关键能力。
（3）端侧覆盖：缺少面向园区管理员的移动管理工具，以及面向 C 端用户的完整商城与会员体系。
（4）数据层面：现有字段体系与新平台存在差异，需要进行系统性的字段映射与历史数据迁移。
（5）外部平台集成：未覆盖美团、抖音两大主流第三方平台，缺乏团购券核销、结算、对账等全链路能力。
因此本项目旨在全新开发一套"袁夫稻田智慧园区综合管理平台"，采用三层架构体系，覆盖 PC 管理后台、园区管理移动端、商户管理小程序与 C 端用户小程序，实现多园区统一运营、数据驱动决策、全链路消费闭环。V2 版本新增美团/抖音第三方平台集成能力。
1.2 项目目标
（1）建成三层架构体系：总部管理后台 → 园区管理层（PC+小程序） → 用户端（C 端小程序）。
（2）支持多园区统一管控：总部可查看全部园区经营数据，亦可一键切换到指定园区、指定商户进行精细化管理。
（3）全链路数字化运营：覆盖店铺管理、用户管理、消费管理、储值充值、活动推送、闸机通行等核心业务场景。
（4）数据驱动决策：提供销售、客流、用户增长、财务等多维度数据看板。
（5）平滑数据迁移：完整梳理原有字段体系，建立字段映射表，确保历史数据无丢失、可追溯地迁移到新平台。
（6）美团全链路集成：实现门店映射、团购券在线核销、T+1 自动结算、对账明细、10% 佣金抽成（V2 新增）。
（7）抖音全链路集成：实现门店映射、团购券/套餐/次卡/代金券/组合券包核销、核券预检、自动结算、对账明细、5% 佣金抽成（V2 新增）。
1.3 文档范围
本文档覆盖以下范围：第一层总部管理后台（PC Web 端）；第二层园区管理层（PC Web 端 + 微信小程序/公众号移动端 + 商户端小程序）；第三层用户层（微信小程序端）；数据迁移字段映射方案；权限模型与账号体系设计；多端 UI/交互规范；美团核销模块（V2 新增）；抖音核销模块（V2 新增）。

二、整体架构设计
2.1 三层架构总览
平台采用"总部 - 园区 - 用户"三层架构，各层职责清晰，数据逐层汇聚，权限逐级下放。

2.2 权限模型（RBAC）
2.2.1 角色定义

| 角色名称 | 角色编码 | 层级 | 权限说明 |
|---------|---------|------|---------|
| 平台超级管理员 | ADMIN_PLATFORM | 总部 | 平台最高权限，所有功能可访问 |
| 总部运营人员 | PLATFORM_OPER | 总部 | 查看所有园区数据，活动推送，系统配置 |
| 园区管理员 | PARK_ADMIN | 园区 | 本园区所有管理权限，含商户/用户/财务/统计/美团/抖音 |
| 园区经理 | PARK_MANAGER | 园区 | 本园区经营数据查看，部分管理操作 |
| 商户管理员 | SHOP_ADMIN | 园区-商户 | 本店铺经营管理，含商品/订单/对账/提现/美团/抖音 |
| 商户收银员 | SHOP_CASHIER | 园区-商户 | 本店铺收银、核销操作（含美团/抖音核销） |
| 普通用户 | MEMBER | 用户 | 注册、充值、点单、支付、活动参与 |

2.2.2 权限维度
（1）数据权限：总部可见全部园区数据，园区管理员仅见本园区数据，商户仅见本店铺数据。
（2）功能权限：按模块划分（店铺/用户/统计/财务/系统配置/美团/抖音 等）。
（3）操作权限：增、删、改、查四级粒度。
（4）视角切换：总部可一键切换到指定园区或商户视角，菜单和数据自动按视角的可见性配置进行裁剪。

2.2.3 视角切换与动态菜单
平台总部人员通过左下角"当前视角"下拉框切换：平台总部视角（全量菜单+全部数据）→ 指定园区视角（园区级菜单+本园区数据）→ 指定商户视角（商户级菜单+本店铺数据）。无权限的菜单自动隐藏，操作按钮置灰并提示。

3.2 详情说明
3.2.1 园区总览（首页）
定位
总部首页，登录后默认展示页面。
页面布局
（1）顶部：全局搜索、消息中心、帮助、设置等图标。
（2）面包屑：智慧园区平台 / 首页。
（3）关键指标卡片：总园区数、总商户数、总用户数、今日总销售额、今日总订单数、今日总客流量、储值池余额。
（4）趋势图表：近 7 天/30 天销售趋势（折线图）、各园区销售额分布（饼图或柱状图）。
（5）园区列表卡片：每个园区的简要经营概况（名称、商户数、今日销售额、本月销售额、状态）。
（6）快捷入口：快速跳转到指定园区管理后台。
特殊说明
通过左下角"当前视角"下拉框，总部人员可一键切换到指定园区或商户的管理视角，查看其完整管理界面，对应菜单与数据自动按权限裁剪。
3.2.2 数据看板
定位为全局数据分析中心，提供多维度数据洞察。
（1）销售统计：订单总数、总销售额、平均客单价，支持选择时间段（今日/本周/本月/自定义）与对比维度（同比、环比）。
（2）客流统计：总客流量、今日客流、高峰时段分布，支持按园区/店铺维度下钻。
（3）用户增长：累计注册用户、新增用户、活跃用户、平均活跃天数。
（4）数据导出：支持看板数据导出为 Excel。
3.2.3 园区管理
（1）园区列表：园区名称、编码、地址、商户数、状态、创建时间，支持搜索与筛选。
（2）新增园区：园区名称、编码、地址、联系人、联系电话、营业时间、LOGO 上传、园区简介。
（3）园区编辑：修改基本信息、启用 / 停用园区。
（4）园区数据快照：一键跳转到该园区视角进行后台管理。
3.2.4 店铺管理
（1）店铺列表：字段含店铺名称、地址、类型（餐饮/零售/体验/其他）、状态（营业中/休息/关闭）、创建时间，可查看店铺二维码。
（2）店铺统计：店铺总数、营业中、休息中、待审核的卡片指标，状态与类型分布图。
（3）店铺类型：新增 / 编辑店铺分类，含名称、编码、描述、状态。
3.2.5 用户管理
（1）用户列表：用户 ID、昵称、手机号、OpenID、注册时间、购买次数、消费金额、充值余额、会员等级、人脸录入状态。支持按昵称、手机号、会员等级、人脸状态筛选；支持手动添加与批量导入用户。
（2）用户统计：总用户数、今日新增、今日活跃、平均活跃天数；用户增长趋势折线图与用户活跃分布图。
（3）充值记录：手机号、金额（充值/消费）、状态（成功/失败/退款）、支付方式、交易号、备注、时间。
3.2.6 消费管理
（1）订单列表：订单号、用户信息、商户名称、商品/金额、支付方式、订单状态、下单时间。状态包括待支付、已支付、已核销、已取消、退款处理中、已退款。
（2）核销管理：扫码核销 + 手动核销，核销记录可追溯。
（3）退款处理：申请列表、审核退款、回款到原支付方式，留存审核备注。
3.2.7 销售统计
销售概览（订单总数、总销售额、平均订单金额）、销售趋势（按周/月/日，可按店铺维度下钻）、储值统计（充值总额、充值用户数、储值消费金额、储值余额池）、数据导出。
3.2.8 客流统计
客流概览（今日/本周/本月，同比环比）、客流趋势折线图、24 小时客流热力图、客流明细。数据来源说明：通过闸机扫码 / 人脸通行自动采集。
3.2.9 财务管理
（1）园区账户：园区总收益、账户余额、冻结金额、线下收入、已提现、待入账、可用余额；账户明细按店铺维度展示。
（2）提现管理：提现申请列表（店铺、金额、银行账号、开户姓名、申请时间、状态、审核时间），支持通过 / 驳回 + 审核备注；提现记录历史。提现审核通过后自动生成对账单（T+1 结算），记录提现流水与余额变动。
（3）对账单：园区与商户之间的对账汇总，支持日期范围筛选、导出。
3.2.10 员工管理
（1）员工列表：用户名、姓名、手机号、性别、所属园区/店铺、角色、岗位、创建时间、在线状态。支持新增、编辑、启用/禁用、重置密码。
（2）岗位管理：岗位名称、编码、所属角色、岗位职责、任职要求、状态。示例：园区管理员、火车餐厅经理、火车餐厅前厅主管、咖啡店长等。
3.2.11 闸机管理
（1）闸机列表：闸机名称、类型、方向（进/出）、设备 SN、状态、创建时间，支持新增、编辑、查看进出记录。
（2）人脸管理：用户 ID、手机号、人脸照片、Agora 特征 ID、状态、创建时间，支持手动录入人脸。
（3）进出记录：用户、闸机、方向、相似度、验证方式（人脸/扫码）、进出时间。
3.2.12 活动管理
活动列表、创建活动（标题、封面、富文本内容、活动时间、适用园区/商户、活动类型）。活动类型涵盖优惠券、满减、限时折扣、节日活动、会员专享。活动效果统计含曝光量、参与人数、转化订单数、核销率。
3.2.13 小程序管理
分为客户端、商户端、管理端三个子模块。各模块管理对应小程序的运营数据（累计用户、日活、储值池、入驻商户、商户端账户数、扫码次数、待处理消息等）、版本与发布信息、首页配置与推送策略等。
3.2.14 权限管理
角色权限二维表，按模块/页面配置可见、可操作权限；角色与员工关联管理；视角切换时菜单与按钮自动按可见性裁剪。
3.2.15 数据迁移管理
（1）旧平台数据库连接配置（支持 MySQL / SQL Server 等）。
（2）字段映射关系配置管理。
（3）迁移任务创建与执行（支持试运行与正式运行）。
（4）迁移结果校验与报告生成。
（5）数据一致性检查与回滚机制。
3.2.16 系统设置
（1）小程序配置：微信小程序 AppID / AppSecret / 回调地址、订阅消息模板配置。
（2）操作日志：用户、操作类型、模块、IP、状态、时间。
3.2.17 美团核销（V2 新增）
定位
集成美团/大众点评平台，实现团购券在袁夫稻田园区内的在线核销、结算、对账全链路管理。
（1）美团数据看板
- 累计核销订单数、累计结算金额、待结算金额、本月新增门店。
- 近 7 天/30 天核销趋势折线图、各门店核销分布柱状图。
- 数据实时同步美团 OpenAPI，T+1 结算。
（2）美团核销记录
- 列表字段：核销编号、团购券名称、用户手机号（脱敏）、核销金额、佣金（10%）、结算金额、核销时间、操作人、所属门店。
- 支持按日期范围、门店、操作人筛选；支持导出 Excel。
（3）美团结算管理
- 结算列表：结算批次号、门店名称、核销笔数、核销总额、佣金总额、结算金额、结算状态（待结算/已结算/已打款）、结算日期。
- 结算规则：T+1 自动结算，美团扣除 10% 佣金后打款至平台账户。
- 支持手动触发结算、结算明细下钻。
（4）美团门店配置
- 门店列表：平台门店名称、美团门店 ID、美团门店名称、授权状态、同步状态。
- 门店授权：通过美团 OAuth 2.0 授权流程，将平台门店与美团门店建立映射关系。
- 4 步开通流程：申请美团合作 → 填写门店信息 → OAuth 授权绑定 → 同步团购券数据。
（5）美团 API 配置
- AppID、AppSecret、回调地址配置。
- 接口调用日志与异常告警。
3.2.18 抖音核销（V2 新增）
定位
集成抖音本地生活平台，实现团购券/套餐/次卡/代金券/组合券包在袁夫稻田园区内的在线核销、结算、对账全链路管理。
（1）抖音数据看板
- 累计核销订单数、累计结算金额、待结算金额、本月新增门店。
- 近 7 天/30 天核销趋势折线图、各门店核销分布柱状图。
- 数据实时同步抖音开放平台 API，T+1 结算。
（2）抖音核销记录
- 列表字段：核销编号、券名称、券类型（团购套餐/次卡/代金券/组合券包）、用户手机号（脱敏）、核销金额、佣金（5%）、结算金额、核销时间、操作人、所属门店。
- 支持按日期范围、门店、券类型、操作人筛选；支持导出 Excel。
- 核销前执行"宣券"预检：检查券状态（是否已使用/过期/退款）、适用门店匹配、使用时间限制。
（3）抖音结算管理
- 结算列表：结算批次号、门店名称、核销笔数、核销总额、佣金总额（5%）、结算金额、结算状态（待结算/已结算/已打款）、结算日期。
- 结算规则：T+1 自动结算，抖音扣除 5% 佣金后打款至平台账户。
- 支持手动触发结算、结算明细下钻。
（4）抖音门店配置
- 门店列表：平台门店名称、抖音门店 POI ID、抖音门店名称、授权状态、同步状态。
- 门店授权：通过抖音开放平台授权流程，将平台门店与抖音 POI 门店建立映射关系。
- 团购券同步：自动拉取抖音侧已上架的团购券/套餐/次卡/代金券/组合券包，建立本地映射。
（5）抖音 API 配置
- Client Key、Client Secret、回调地址配置。
- 接口调用日志与异常告警。
（6）抖音券类型定义
- 团购套餐：固定商品组合，一次性核销。
- 次卡：可分多次核销的计次券，每次核销扣减次数。
- 代金券：定额抵用券，核销时抵扣指定金额。
- 组合券包：多张券打包销售，支持分别核销。

6.1 迁移原则

七、交互与体验设计原则
7.1 设计风格 · 自然田园风
品牌调性：袁夫稻田的品牌核心是"自然、田园、温暖、品质"。整个平台设计围绕这一调性展开，采用温暖大地色系与稻田绿色系的搭配。
色彩体系：主色 #2D5016（深稻绿），辅助色 #5B8C2A（稻苗绿），暖色 #8B5E3C（大地棕）、#C8963E（稻穗金），背景色 #F5F0E8（米白）、#E8D5B7（稻杆色）。
字体规范：中文使用思源宋体（标题）、思源黑体（正文）；字号层级 32 / 20 / 16 / 14 / 12px。
UI 元素：圆角 8 - 12px、线性图标 2px 描边、柔和阴影 rgba(0,0,0,0.08)、充足留白。
7.2 响应式与适配要求
（1）PC 端：支持 1920×1080 及以上分辨率，兼顾 1440×900。
（2）小程序：遵循微信小程序设计规范，基础库版本 ≥ 3.0；适配主流手机机型。
7.3 交互规范
（1）数据加载：骨架屏 + 分步加载（优先展示框架数据）。
（2）操作反馈：Toast 提示 + 成功 / 失败状态明确标识。
（3）列表翻页：支持分页（10 / 20 / 50 条/页）+ 滚动加载。
（4）时间选择：提供时间范围快捷选择（今日 / 本周 / 本月 / 自定义）。
（5）权限控制：无权限的菜单自动隐藏，操作按钮置灰并提示。
（6）支付安全：亮码支付二维码动态刷新；金额确认弹窗；支付密码可选。
